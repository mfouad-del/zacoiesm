# خطة تحويل وتطوير مشروع IEMS Pro

تم تحويل المشروع بنجاح ليعمل بمعمارية منفصلة (Frontend & Backend) لتحقيق أقصى قدر من الأداء والمرونة.

## 1. الهيكلة الجديدة (Architecture)

*   **الواجهة الأمامية (Frontend):**
    *   **التقنية:** React + Vite (سريع جداً وخفيف).
    *   **الاستضافة:** Bluehost (يتم رفع الملفات الثابتة).
    *   **المسار:** المجلد الرئيسي للمشروع.
*   **الواجهة الخلفية (Backend):**
    *   **التقنية:** Node.js + Express.
    *   **الاستضافة:** Render.com.
    *   **المسار:** مجلد `backend/`.
*   **قاعدة البيانات:** Supabase (تم الحفاظ عليها).

## 2. تعليمات التشغيل والتطوير

### أولاً: الواجهة الخلفية (Backend)
هذا هو "العقل" المدبر للمشروع، ويجب تشغيله أولاً.

1.  انتقل لمجلد الباك اند: `cd backend`
2.  قم بتثبيت المكتبات: `npm install`
3.  أنشئ ملف `.env` داخل مجلد `backend` وضع فيه بيانات Supabase:
    ```env
    SUPABASE_URL=رابط_سوبابيس_الخاص_بك
    SUPABASE_ANON_KEY=مفتاح_سوبابيس_الخاص_بك
    PORT=3001
    ```
4.  لتشغيله محلياً: `npm start`
5.  **للنشر على Render:**
    *   اربط المستودع (Repo) بـ Render.
    *   اختر `Root Directory` ليكون `backend`.
    *   أضف متغيرات البيئة (Environment Variables) في لوحة تحكم Render.

### ثانياً: الواجهة الأمامية (Frontend)
هذه هي الشاشات التي يراها المستخدم.

1.  تأكد من وجود ملف `.env` في المجلد الرئيسي يحتوي على رابط الباك اند:
    ```env
    VITE_API_URL=https://zacoiesm.onrender.com/api
    ```
    *(في حالة التطوير المحلي استخدم: http://localhost:3001/api)*
2.  لتشغيل المشروع للتطوير: `npm run dev`
3.  **للنشر على Bluehost:**
    *   نفذ الأمر: `npm run build`
    *   سينتج مجلد اسمه `dist`.
    *   قم بضغط محتويات هذا المجلد ورفعها إلى `public_html` في استضافة Bluehost.

## 3. تغيير الهوية البصرية (Logo & Favicon)

لتغيير اللوجو والأيقونة، قم ببساطة باستبدال الملفات التالية في المجلد الرئيسي بنفس الأسماء:
1.  `icon.svg` (أيقونة المتصفح)
2.  `logo.png` (شعار الموقع)

## 4. التطويرات التي تمت
*   **فصل الكود:** تم عزل كود السيرفر عن كود الواجهة.
*   **تنظيف المشروع:** تم حذف ملفات Next.js غير المستخدمة لتقليل حجم المشروع.
*   **ربط API:** تم تعديل `App.tsx` ليجلب البيانات من السيرفر بدلاً من الاعتماد الكلي على الذاكرة المحلية.

## 5. الخطوات القادمة المقترحة
*   تطوير المزيد من نقاط الاتصال (API Endpoints) في `backend/index.js` لتغطية كافة خصائص النظام.
*   تفعيل نظام المصادقة (Auth) بشكل كامل بين الفرونت اند والباك اند.
